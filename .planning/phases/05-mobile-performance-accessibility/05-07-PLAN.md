    Two surgical additions:

    **(A) `src/components/DisciplineCard.astro`:** Add an inline `<script>` block (Astro auto-inlines into the bundle — no `client:*` directive needed) implementing the IntersectionObserver pattern from 05-RESEARCH §3.5 step 2 + 05-UI-SPEC §"Touch entrance shimmer":

    ```
    <script>
      const isTouchDevice = window.matchMedia('(hover: none)').matches;
      const prefersReduced = window.matchMedia('(prefers-reduced-motion: reduce)');

      function enableEntranceShimmer() {
        const observer = new IntersectionObserver((entries) => {
          entries.forEach((entry) => {
            if (entry.isIntersecting) {
              entry.target.classList.add('is-entered');
              observer.unobserve(entry.target);
            }
          });
        }, { threshold: 0.4 });
        document.querySelectorAll('.b-card, .b-bio').forEach((el) => observer.observe(el));
      }

      if (isTouchDevice && !prefersReduced.matches) {
        enableEntranceShimmer();
      }

      prefersReduced.addEventListener?.('change', (e) => {
        if (!e.matches && isTouchDevice) enableEntranceShimmer();
      });
    </script>
    ```

    The script lives ONCE — put it in DisciplineCard.astro since that's the primary subject; the `.b-bio` selector still matches if it appears on the same page. (Alternative: put it in Base.astro for site-wide; but DisciplineCard is fine since only index.astro renders both `.b-card` and `.b-bio`.)

      50%  { box-shadow: 0 0 0 6px color-mix(in oklab, currentColor 8%, transparent); }

      .b-card.is-entered,

      }
