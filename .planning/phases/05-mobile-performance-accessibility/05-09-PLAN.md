### Task 1 — Carousel dot semantics + hit target (closes aria-allowed-attr, aria-required-children, target-size, label-content-name-mismatch — 27 weighted points)

<read_first>
- /Users/caleb/projects/personal-website/src/pages/index.astro lines 86-100 (markup) and 298-322 (CSS) and 685-720 (JS that toggles `.is-active` + `aria-selected`)
- /Users/caleb/projects/personal-website/.planning/phases/05-mobile-performance-accessibility/05-UI-SPEC.md §Visual Contract for any explicit carousel-dot semantics lock
</read_first>

<action>
Drop the `role="tablist"` + `role="tab"` semantics entirely — the carousel is a dot navigator, not a real tab set (no `tabpanel` siblings exist; the portrait image swap is announced via the surrounding `<div class="b-portrait">`). Use a simpler ARIA pattern:

- `<div class="bp-dots">` — remove `role="tablist"` and `aria-label="Portrait photo selector"`. Wrap the buttons in a `<div role="group" aria-label="Portrait photo selector">` or drop the role entirely (allowed since the buttons are self-describing).
- Each `<button class="bp-dot">`: remove `aria-selected`. Replace with `aria-current="true"` on the active dot only (omitted on others). Keep `type="button"`, keep `aria-label="Photo N"`.
- Hit target: bump CSS to `width: 24px; height: 24px; padding: 8px; background: transparent;`. Move the visible 8×8 dot to a `::before` pseudo with `content: ''; display: block; width: 8px; height: 8px; border-radius: 50%; background: rgba(242, 235, 219, 0.45);`. The `.is-active::before` rule swaps `background: var(--paper); transform: scale(1.25);`. This keeps the visual identical (8×8 dot) but doubles the tap surface (24×24).
- `.bp-dot:focus-visible` — focus ring stays on the button (outer 24×24), still 2px outline.
- Gap between dots: keep `.bp-dots { gap: 6px; }` — total dot center-to-center is now ~30px, well above the 24px target-spacing floor.
- Carousel JS (`/data-bp-dot` handlers near line 689): replace `aria-selected` toggle with `aria-current` toggle. The `dot.setAttribute('aria-selected', ...)` lines flip to `dot.setAttribute('aria-current', i === activeIndex ? 'true' : 'false')` (or use `removeAttribute('aria-current')` on inactive — cleaner).
- Don't change keyboard handling — buttons remain Tab-reachable + Enter/Space activatable.
</action>
