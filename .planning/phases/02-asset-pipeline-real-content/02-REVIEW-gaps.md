# Phase 2 Gap-Closure Code Review

**Reviewed:** 2026-05-11
**Depth:** standard
**Scope:** commits `dd86465..HEAD` touching `scripts/pdf-preprocess.mjs` and `scripts/verify-build.sh` (plans 02-05, 02-06, 02-07).
**Status:** issues_found

## Summary

Gap closure largely lands the original BLOCKER (CR-01) and WARNINGs (WR-01, WR-02) cleanly, and Gate 13 provides solid runtime regression coverage for the draft-skip behavior. However, the patches focused on the *prebuild path* and missed the *cross-build orphan path*: assets generated by an earlier build (when a piece was non-draft, or before a piece was deleted, or when `fullPdf` was set and is now omitted) are not pruned. This recreates the spirit of CR-01 — stale draft assets ship — through a different door. It is the same class of bug the plan claimed to close.

The new Gate 12 sub-gates and Gate 13 are mostly robust on macOS BSD tools, but have several quoting and regex precision issues that will produce false negatives (a dirty repo passes the gate) rather than false positives. False negatives are the worse failure mode for a smoke gate whose job is to fail loudly.

## Critical Issues

### CR-01-GAPS: Stale per-slug assets leak across builds — same class as the original CR-01

**File:** `scripts/pdf-preprocess.mjs:55-97, 130-243`
**Issue:** The CR-01 fix at `discoverPieces` (line 85) correctly skips draft pieces during the *current* prebuild. But `pdf-preprocess.mjs` only ever **adds or overwrites** files under `public/generated/pdf-thumbs/<slug>/` and `public/source-pdfs/<slug>.pdf`. It never prunes a slug-directory it no longer recognises. Three concrete leak paths reintroduce the original CR-01 failure mode (draft assets on disk → committed via D-03 → ship to production):

1. **Draft flip:** Caleb sets `draft: false` on `finance-real-piece`, runs build, gets `public/generated/pdf-thumbs/finance-real-piece/cover.webp` + `public/source-pdfs/finance-real-piece.pdf`. Later flips `draft: true` (the exact scenario plan 02-07 just executed in commit `3efeba4`). `discoverPieces` now SKIPs the piece — but the previously-generated thumb dir and source PDF remain on disk and ship. CR-01 says "draft pieces must NOT rasterize or copy"; in steady state they still ship.
2. **Piece rename / delete:** rename `personal-foo` → `personal-bar`. New slug rasterizes; old `pdf-thumbs/personal-foo/` and `source-pdfs/personal-foo.pdf` orphan.
3. **`fullPdf` removed:** piece had `fullPdf: "/source-pdfs/<slug>.pdf"`, now it's removed from frontmatter (recruiter-distracting "Open full PDF" link no longer wanted). Cache hit path skips `copySourcePdf` (line 142, `if (fullPdf)`), but the previous `public/source-pdfs/<slug>.pdf` is not deleted. Stale PDF still downloadable via direct URL even though the link is gone — same "is this still relevant?" leak vector.

This mirrors WR-01 exactly (orphan `page-N.webp` within a piece dir) but at the slug-directory level, and it mirrors CR-01 exactly (draft assets shipping) for case (1). The WR-01 fix (lines 170-184) is intra-piece-only; it cannot see siblings.

**Fix:** Add a top-level orphan-prune step in `main()` after `discoverPieces`, before the per-piece loop. Build the set of expected slugs (non-draft pieces with `source.pdf`), then walk `OUTPUT_DIR` and `SOURCE_PDF_DIR`, removing anything not in that set. Suggested:
```javascript
async function pruneOrphanSlugs(activeSlugs) {
  // OUTPUT_DIR: each child is a slug directory
  for (const dir of [OUTPUT_DIR]) {
    const entries = await fs.readdir(dir).catch(() => []);
    for (const entry of entries) {
      if (!activeSlugs.has(entry)) {
        await fs.rm(path.join(dir, entry), { recursive: true, force: true });
        console.log(`PRUNE-ORPHAN ${entry}/ (slug no longer active — draft, deleted, or missing source.pdf)`);
      }
    }
  }
  // SOURCE_PDF_DIR: each child is <slug>.pdf
  const pdfs = await fs.readdir(SOURCE_PDF_DIR).catch(() => []);
  for (const pdf of pdfs) {
    const slug = pdf.replace(/\.pdf$/, '');
    if (!activeSlugs.has(slug)) {
      await fs.unlink(path.join(SOURCE_PDF_DIR, pdf));
      console.log(`PRUNE-ORPHAN ${pdf} (slug no longer has fullPdf or is now draft)`);
    }
  }
}

async function main() {
  const pieces = await discoverPieces();
  const activeSlugs = new Set(pieces.map((p) => p.slug));
  await pruneOrphanSlugs(activeSlugs);
  // Also: prune orphan source-pdf for pieces whose fullPdf is now unset
  const fullPdfSlugs = new Set(pieces.filter((p) => p.fullPdf).map((p) => p.slug));
  // ...prune SOURCE_PDF_DIR using fullPdfSlugs instead of activeSlugs
  // ... rest of main loop
}
```
And add a Gate 14 in `verify-build.sh` that asserts `public/generated/pdf-thumbs/<dir>/` set ⊆ non-draft-piece-slug set (analogous to Gate 13's drift-detection ethic).

## Warnings

### WR-01-GAPS: Gate 12 draft-detection regex misses indented or quoted YAML

**File:** `scripts/verify-build.sh:250` (`list_non_draft_pieces`) and line 305 (Gate 12d's PLACEHOLDER scan; line 325 banned-phrase scan inherits the same)
**Issue:** `grep -q '^draft: true' "$f"` only matches the literal byte sequence `draft: true` at column 1. It will MISS:

- `draft: "true"` (YAML lets you quote booleans; gray-matter parses that as `"true"` string, not boolean — but Astro's Zod schema declares `z.boolean()` so it'd fail validation; still, the grep is brittle)
- Whitespace before `draft:` (illegal at the top frontmatter level but valid in nested keys; should not match anyway)
- `draft:true` (no space — uncommon but valid YAML)
- Trailing spaces / CRLF: `draft: true \r\n` — actually matches because `^draft: true` anchors to start, not end. Confirmed via test.

Inverse risk: a piece with `draft: true` in the BODY (e.g., a paragraph starting at column 0 saying "draft: true was a mistake") would be falsely classified as draft. Unlikely but possible — `grep -q '^draft: true' "$f"` scans the whole file, not just the frontmatter block.

**Fix:** Either (a) parse frontmatter properly via the same `python3` heredoc pattern Gate 10/11 use, or (b) tighten the grep to `^draft:\s*true\s*$` and scope it to the YAML block via `sed -n '/^---$/,/^---$/p'`:
```bash
list_non_draft_pieces() {
  local f
  for f in src/content/pieces/*/index.md; do
    [[ -f "$f" ]] || continue
    sed -n '/^---$/,/^---$/p' "$f" | grep -qE '^draft:[[:space:]]*true[[:space:]]*$' && continue
    echo "$f"
  done
}
```

### WR-02-GAPS: Gate 13 EXIT trap fires before final `trap - EXIT`, overwriting any subsequent ERR/EXIT trap from later code additions

**File:** `scripts/verify-build.sh:357, 433-434`
**Issue:** `trap cleanup_draft_test EXIT` is set, then later `trap - EXIT` clears it. Between those two lines, ANY future code added (the success/failure echo at 437-443 is fine, but any caller-added gate inserted in the middle would inherit the cleanup trap unexpectedly). Worse: if Gate 13's body throws under `set -e` (e.g., `mkdir -p` permission denied on `src/content/pieces/`), the EXIT trap fires → cleanup runs → exits with the failing line's code. That's actually the correct behavior, but the explicit `cleanup_draft_test` call at line 433 followed by `trap - EXIT` at line 434 is brittle: if a future maintainer adds a gate between lines 434 and 437, that gate runs WITHOUT cleanup protection.

Less hypothetical: the trap's cleanup uses `2>/dev/null || true` (line 354). If `rm -rf` fails (permission, FS busy), it silently swallows the error — the next run hits the pre-clean (line 360) which has the same swallowing — and if THAT also fails, `mkdir -p` at line 363 may then fail unexpectedly under `set -e` because the directory exists but as something un-writable.

**Fix:** Move the `trap - EXIT` to immediately after the assertions, INSIDE Gate 13's scope, so the trap only ever covers Gate 13's body. Drop the explicit cleanup_draft_test call since the trap now fires reliably. Also: log (not swallow) cleanup failures so a stuck fixture is visible:
```bash
cleanup_draft_test() {
  rm -rf "$DRAFT_TEST_DIR" "$DRAFT_TEST_THUMB_DIR" "$DRAFT_TEST_SOURCE_PDF" \
    || echo "  WARN: Gate 13 cleanup failed for one or more of: $DRAFT_TEST_DIR $DRAFT_TEST_THUMB_DIR $DRAFT_TEST_SOURCE_PDF" >&2
}
```

### WR-03-GAPS: Gate 13 SIGKILL / SIGSEGV pollution leaks fixture into source tree, breaking next `astro build`

**File:** `scripts/verify-build.sh:348-433`
**Issue:** The trap at line 357 only fires on EXIT (not on uncatchable signals: SIGKILL, SIGSEGV, OOM-killer, hard reboot). If `verify-build.sh` is `kill -9`'d while Gate 13 has the fixture in place, `src/content/pieces/__draft-skip-test__/` survives. Next `npm run build` will:

1. Astro content-collection loader (`src/content.config.ts:6-25`) globs `**/index.md` and applies the Zod schema to the fixture.
2. Schema requires `hero: image()` which validates against an actual decodable image. The Gate 13 stub is `printf '\x89PNG\r\n\x1a\n'` (8 bytes — PNG magic only, no IHDR chunk). Sharp will reject this.
3. `pdf-preprocess.mjs` will try to open the stub `source.pdf` (`echo > source.pdf` = 1-byte newline). pdfjs will throw "Invalid PDF structure" — but discoverPieces still skips it because `draft: true`. So the prebuild path is safe.
4. Astro build still fails on the schema validation in step 2.

Net: a hard-killed verify-build run wedges the next build. The pre-clean at line 360 catches the soft case (next `verify-build.sh` invocation cleans first), but does NOT help if the user runs `npm run build` directly without `verify-build.sh` in between.

**Fix:** Either (a) make the fixture self-cleaning at the START of `pdf-preprocess.mjs` itself (not great — pollutes prod code with test concerns), (b) move Gate 13 OUT of the source tree by setting up the fixture in a temp dir and pointing pdf-preprocess at a `--pieces-dir=` flag (requires script refactor), or (c) pragmatic minimum: add a top-level pre-clean for the fixture slug at the very start of `verify-build.sh`, before any gate runs:
```bash
# Top of script, after `set -euo pipefail`:
rm -rf "src/content/pieces/__draft-skip-test__" \
       "public/generated/pdf-thumbs/__draft-skip-test__" \
       "public/source-pdfs/__draft-skip-test__.pdf" 2>/dev/null || true
```
Option (b) is the right long-term fix; option (c) is the 2-minute version that closes the immediate hole.

### WR-04-GAPS: `set -e` interaction with `grep -v '^#' | grep -iE '...' || true` swallows pipe-failure exit codes other than grep's no-match

**File:** `scripts/verify-build.sh:325`
**Issue:** `hit=$(grep -v '^#' "$np_file" 2>/dev/null | grep -iE 'passionate|...' || true)`. The `|| true` swallows ANY non-zero from the pipeline, including `grep: $np_file: Permission denied` or out-of-memory. The intent is clearly "tolerate no-match (exit 1)" but the actual semantic is "tolerate any failure". A permissions issue on a piece file would silently report "no banned phrases" — a false-negative pass on a gate whose entire purpose is to fail loudly.

Same pattern at line 305 (`grep -l 'PLACEHOLDER' "$np_file" >/dev/null 2>&1`) — `2>/dev/null` swallows stderr; the `if grep -l ... >/dev/null 2>&1` is fine because it's exit-code-checked in an `if`, but a subtle distinction.

Also: `pipefail` is set at line 6, so if the FIRST grep (`grep -v '^#'`) fails non-zero (impossible for `-v` unless the file doesn't exist), the `|| true` masks it. With `pipefail` enabled, the pipe's exit is the rightmost non-zero, but `|| true` still wins. Net: gates can silently false-negative.

**Fix:** Be explicit about which exit codes mean "no match" vs "real error":
```bash
hit=$(grep -v '^#' "$np_file" | grep -iE 'passionate|multidisciplinary|intersection of')
gr=$?
if (( gr != 0 && gr != 1 )); then
  echo "  FAIL: Gate 12e — grep error ($gr) scanning $np_file"
  fail=1
  continue
fi
```

### WR-05-GAPS: Gate 13 fixture `fullPdf` references canonical href but rasterization never runs — assertion-by-omission risk

**File:** `scripts/verify-build.sh:371` and `scripts/pdf-preprocess.mjs:116-128`
**Issue:** The fixture has `fullPdf: "/source-pdfs/__draft-skip-test__.pdf"` — but because the draft-skip fires in `discoverPieces` (line 85) BEFORE `rasterizePiece` (and therefore before `copySourcePdf`), the WR-02 contract assertion (line 119) is never exercised against the fixture. Assertion 4 (line 421) confirms the source PDF is NOT copied, which is correct CR-01 behavior, but it does NOT prove the WR-02 contract still works for the non-draft path. Gate 13 only covers CR-01, not WR-02 regressions.

This is partially an INFO-tier concern (the gate's name `CR-01 draft-skip smoke check` is honest about scope), but the fixture's `fullPdf` line is misleading — a reader assumes it's testing the WR-02 path too. Either drop `fullPdf` from the fixture (it's a no-op given the skip fires first) or add a Gate 14 that exercises the non-draft path with a deliberately wrong `fullPdf` value and asserts the contract throws.

**Fix:** Either remove `fullPdf` from the fixture frontmatter (it's load-bearing comment, not load-bearing assertion):
```yaml
# In Gate 13 fixture: drop fullPdf and pdfPaginate; they're not exercised because draft-skip fires first.
```

## Info

### IN-01-GAPS: pdfPaginate JSON.stringify in hash is order-sensitive

**File:** `scripts/pdf-preprocess.mjs:104`
**Issue:** `JSON.stringify(pdfPaginate ?? [])` hashes by array order. Reordering `[1, 5, 12]` → `[1, 12, 5]` produces a different hash → cache miss → full re-render even though render output (set of files) is identical. Wasted work on Caleb editing a deck without changing the page set.
**Fix:** Hash the sorted unique set: `JSON.stringify([...new Set(pdfPaginate ?? [])].sort((a, b) => a - b))`. Also dedupes (see IN-02).

### IN-02-GAPS: pdfPaginate duplicates cause double-render

**File:** `scripts/pdf-preprocess.mjs:162`
**Issue:** `pagesToRender = [1, ...((pdfPaginate ?? []).filter((n) => n !== 1))]`. If pdfPaginate is `[1, 5, 5, 12]`, result is `[1, 5, 5, 12]` — page 5 renders twice (writes to the same file twice — correct output but wasted work). Schema doesn't enforce uniqueness either.
**Fix:** `pagesToRender = [...new Set([1, ...(pdfPaginate ?? [])])]` — covers dedupe + page-1 hoisting in one expression.

### IN-03-GAPS: Gate 12c category regex won't match YAML quoted values

**File:** `scripts/verify-build.sh:285`
**Issue:** `^category: ${required_cat}( |$)` does not match `category: "design"` or `category: 'design'`. Caleb's current real pieces use unquoted (verified in source), so it works today, but YAML allows both. A future copy-paste from another piece's frontmatter that quoted the value would silently miss the strong-floor count.
**Fix:** Allow optional surrounding quote: `^category:[[:space:]]*['"]?${required_cat}['"]?[[:space:]]*$` — and scope to frontmatter via `sed -n '/^---$/,/^---$/p'`.

### IN-04-GAPS: Gate 13 fixture's `EXISTING_HERO` may copy a `.webp` to `hero.png`

**File:** `scripts/verify-build.sh:381-386`
**Issue:** `find ... -name 'hero.*'` matches `hero.webp` too (current real pieces use `.webp` per `ls`). The script then `cp $EXISTING_HERO $DRAFT_TEST_DIR/hero.png` — copying WebP bytes into a `.png` file. Sharp would reject this if Astro ever validated it (which only happens if the fixture leaks per WR-03-GAPS). Cosmetically wrong; functionally invisible because draft-skip prevents validation. Worth fixing for clarity.
**Fix:** Detect the source extension and preserve it, or just use the 8-byte PNG-magic stub unconditionally:
```bash
# Always use stub — it's enough since pdf-preprocess never opens it under draft-skip.
printf '\x89PNG\r\n\x1a\n' > "$DRAFT_TEST_DIR/hero.png"
```
