### `scripts/pdf-preprocess.mjs` (NEW — build-script, file-I/O + transform)

**Analog:** `scripts/pdf-poc.mjs` (the entire file is the seed pattern). Phase 2 extends — does NOT rewrite from scratch.

**Imports pattern** (`scripts/pdf-poc.mjs:7-8`):
```javascript
import fs from 'node:fs';
import { getDocument } from 'pdfjs-dist/legacy/build/pdf.mjs';
```
Phase 2 extension: switch `node:fs` to `node:fs/promises`, add `node:path`, `node:crypto.createHash`, `sharp`, and `gray-matter`. Keep the verbatim `pdfjs-dist/legacy/build/pdf.mjs` entry — it's the load-bearing line.

**CMap / standard-font URL constants** (`scripts/pdf-poc.mjs:10-11`):
```javascript
const CMAP_URL = './node_modules/pdfjs-dist/cmaps/';
const STANDARD_FONT_DATA_URL = './node_modules/pdfjs-dist/standard_fonts/';
```
Copy verbatim. Pitfall 3 in RESEARCH.md flags that omitting these makes non-Latin glyphs render as boxes.

**Core rasterization pattern** (`scripts/pdf-poc.mjs:24-45`):
```javascript
const data = new Uint8Array(fs.readFileSync(inputPath));

const pdfDocument = await getDocument({
  data,
  cMapUrl: CMAP_URL,
  cMapPacked: true,
  standardFontDataUrl: STANDARD_FONT_DATA_URL,
}).promise;

const page = await pdfDocument.getPage(1);
const canvasFactory = pdfDocument.canvasFactory; // built-in NodeCanvasFactory in pdfjs 5.x
const viewport = page.getViewport({ scale: 1.0 });
const canvasAndContext = canvasFactory.create(viewport.width, viewport.height);

await page.render({
  canvasContext: canvasAndContext.context,
  viewport,
}).promise;

const image = canvasAndContext.canvas.toBuffer('image/png');
fs.writeFileSync(outputPath, image);

page.cleanup();
```
**This block is the contract.** Phase 2 wraps it in: (a) a per-piece loop over `[1, ...pdfPaginate]`, (b) Sharp resize+webp encode replacing the direct PNG write, (c) a `pdfDocument.cleanup()` call after the page loop. Bump `scale: 1.0` → `scale: 2.0` (RESEARCH.md `RENDER_SCALE`, 2x for downsample headroom).

**Error / exit pattern** (`scripts/pdf-poc.mjs:16-20, 47-60`):
```javascript
if (!fs.existsSync(inputPath)) {
  console.error(`POC input not found: ${inputPath}`);
  console.error(`Caleb supplies samples/poc-input.pdf — see Phase 1 CONTEXT D-05.`);
  process.exit(2);
}
// ...
} catch (err) {
  console.error('POC FAIL:', err);
  process.exit(1);
}
```
Phase 2 keeps the same shape: explicit `process.exit(N)` codes (0 ok, 1 generic fail, 2 missing input), `console.error` for diagnostics, `console.log` for `OK ...` lines per output. The phase-2 production script also wants `console.warn` for soft failures (e.g. `pdfPaginate` references a page beyond `numPages` — RESEARCH.md Pitfall 4).

**Hash-cache extension** (no codebase analog — copy from RESEARCH.md Example 1, lines 505–528 of `02-RESEARCH.md`):
```javascript
async function hashInputs(pdfPath, pdfPaginate) {
  const bytes = await fs.readFile(pdfPath);
  return createHash('sha256')
    .update(bytes)
    .update('|paginate=')
    .update(JSON.stringify(pdfPaginate ?? []))
    .digest('hex');
}
```
Per RESEARCH.md A7: include a pipeline-version constant in the hash (`.update('|v=2')`) so script edits invalidate the cache.

**Frontmatter discovery** (no codebase analog — RESEARCH.md Example 1, lines 488–503):
The pre-build script CANNOT use `astro:content` `getCollection` — that API only works inside Astro's runtime. Read the filesystem directly with `fs.readdir(PIECES_DIR)`, parse each `index.md` with `gray-matter`. Pattern excerpt from RESEARCH.md ~line 489.

**Loop-over-files idiom** (`scripts/verify-build.sh:62-72`):
```bash
while IFS= read -r html; do

  grep -q 'Context' "$html" || missing="$missing Context"
  grep -q 'Role' "$html" || missing="$missing Role"
```

Reuse this `while/find` idiom for: gate that checks every paginated piece's HTML contains the expected `<img src="/generated/pdf-thumbs/...">` tags (PIECE-04 validation), and gate that checks every `fullPdf` piece's HTML contains the expected `<a href=...>` (PIECE-06 validation).

**Exit-summary idiom** (`scripts/verify-build.sh:74-81`):

```bash
  echo "FAILED"
  exit 1
```

**Apply to:** `pdf-preprocess.mjs`, all check-* scripts, verify-build.sh extensions.

```javascript
process.exit(0);  // success — required by npm prebuild lifecycle to allow `build` to follow
```


### Diagnostic console pattern
**Source:** `scripts/pdf-poc.mjs` + `scripts/verify-build.sh`

| Use case | Stream | Prefix |
