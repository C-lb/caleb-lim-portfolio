<objective>
Close the three pipeline-correctness findings from 02-REVIEW.md before any real content with source.pdf lands. Currently latent because zero pieces have source.pdf, but each becomes a real production bug the moment Caleb commits a draft piece (CR-01) or shrinks a pdfPaginate array (WR-01) or has a frontmatter typo in fullPdf (WR-02).

Three fixes, all in scripts/pdf-preprocess.mjs:

1. **CR-01 (BLOCKER):** discoverPieces() reads fm.pdfPaginate / fm.fullPdf without checking fm.draft. Every piece with source.pdf gets rasterized to public/generated/pdf-thumbs/ and (with fullPdf) the source PDF copied to public/source-pdfs/. Astro publishes everything under public/ verbatim. Meanwhile getStaticPaths in [category]/[slug].astro:9 filters draft !== true so the HTML detail page is hidden — but the assets are not. A draft piece with predictable slug is fetchable directly. Fix: filter drafts in discoverPieces() before rasterization, with a visible SKIP <slug> (draft) log line.

2. **WR-01:** Cache-miss regeneration overwrites cover.webp + writes new page-N.webp files but never prunes stale page-N.webp files for pages no longer in pdfPaginate. Shrinking [1,5,12,23] → [1,5] leaves page-12.webp and page-23.webp in public/ (committed via D-03, ship to production as dead bytes that can leak intent). Fix: before writing new pages on the regenerate path, read existing thumb directory contents and unlink any page-*.webp not in the expected set (computed from pagesToRender).

3. **WR-02:** No contract between frontmatter fullPdf string and the actual copied file path. copySourcePdf always writes to public/source-pdfs/<slug>.pdf, but the template's <a href={fullPdf} download> uses whatever string is in frontmatter. fullPdf: "/files/wrong.pdf" is a silent 404. Fix: derive the URL from slug at the same path the script writes (preferred per code review's "Option (a) is simpler"), so the schema field becomes effectively a toggle. Note: the template already imports slug — this is mostly a cleanup that aligns intent with implementation. The schema can stay as z.string().optional() to preserve compatibility with any in-flight frontmatter; the script emits the file at the canonical path regardless. Template change is OUT OF SCOPE for this plan (Plan 03 owns the template; if WR-02 needs a template tweak it goes in plan 02-08, but the current template already works because no piece sets fullPdf yet — the fix here is to enforce that the script's output PATH is the canonical path so that any downstream consumer that derives href from slug will succeed).

This plan is the prerequisite for 02-05 (real content). It runs first so all three fixes are in the pipeline BEFORE Caleb commits any source.pdf — preventing the latent BLOCKER from ever firing in production.

Purpose: Close CR-01 + WR-01 + WR-02 from 02-REVIEW.md.
Output: Fixed scripts/pdf-preprocess.mjs that defends against draft-leak, orphan accumulation, and silent 404s.
</objective>
